يهدف هذا النظام إلى أتمتة عمليتين أساسيتين:

1. **جمع وتسجيل بيانات العملاء** — بواسطة الموظفين (الوكلاء الميدانيين)

## 🔷 المرحلة الأولى — تسجيل بيانات العميل الأساسية

### الهدف

تمكين الموظف (الوكيل) من إدخال بيانات العملاء بطريقة سلسة وسريعة لبناء قاعدة بيانات موحدة وموثوقة.

|1|**الاسم الكامل**     |نص   |الاسم الرباعي إن أمكن           |
|2|**رقم الهاتف**       |رقم  |الرقم الأساسي + رقم بديل اختياري|
|3|**العنوان**          |نص   |المدينة / الحي / الشارع         |

### تدفق العمل المقترح (Flow)

```
الوكيل يبدأ المحادثة مع العميل
        ↓
يفتح نموذج التسجيل على جهازه (تطبيق / ويب)
        ↓
يُدخل البيانات حقلاً حقلاً مع التحقق الفوري
        ↓
مراجعة سريعة قبل الحفظ
        ↓
تأكيد الحفظ ← يُرسل إشعار للعميل (SMS / واتساب) اختياري
        ↓
تُضاف السجل إلى قاعدة البيانات المركزية
```

## 🔷 المرحلة الثانية — نظام الضمان المتبادل (Mutual Guarantee)

### المفهوم

نظام يربط **3 عملاء بالضبط** معاً بحيث يضمن كل فرد في المجموعة باقي الأعضاء. تُستخدم هذه الآلية في التمويل الجماعي والقروض الاجتماعية.

### شروط المجموعة

- عدد أعضاء المجموعة: **3 عملاء (ثابت)**
- يجب أن يكون كل عضو مسجلاً في قاعدة البيانات الأساسية أولاً
- لا يمكن حفظ المجموعة إلا باكتمال الأعضاء الثلاثة

## 🔷 المرحلة الثالثة — مقترحات تطوير النظام

### 1. التحقق والجودة

- منع تسجيل نفس رقم الهاتف / رقم الهوية مرتين
- التحقق من صحة تنسيق الهاتف والبريد الإلكتروني
- إلزامية الحقول الأساسية مع اختيارية الحقول الثانوية

### 2. صلاحيات المستخدمين

```
مدير النظام     → إضافة / تعديل / حذف / تقارير كاملة
موظف أول       → إضافة / تعديل سجلاته فقط
مشرف           → مراجعة جميع السجلات + تقارير
قراءة فقط      → عرض البيانات دون تعديل
```

### 3. التقارير والإحصاءات

- عدد العملاء المسجلين (يومي / شهري)
- عدد مجموعات الضمان النشطة
- إجمالي قيم المرتبات المسجلة
- العملاء غير المكتملة بياناتهم
- أداء كل موظف (عدد التسجيلات)

### 4. التكامل مع أنظمة أخرى

- ربط مع نظام الـ ERP (ERPNext) لاستيراد / تصدير البيانات
- إشعارات واتساب / SMS عند اكتمال التسجيل
- تصدير البيانات بصيغة Excel / PDF
