## 🗄️ قاعدة البيانات والذاكرة المؤقتة

|الأداة                         |الوظيفة                                       |الاستخدام المُوصى|
|-------------------------------|----------------------------------------------|----------------|
|**MySQL / PostgreSQL**         |قاعدة البيانات الأساسية للبيانات الدائمة      |✅ أساسي         |
|**Redis**                      |تخزين مؤقت سريع (جلسات، كاش، Queues)          |✅ موصى به       |
|**Memcached**                  |تخزين مؤقت بسيط وخفيف                         |⚠️ بديل لـ Redis |
|**Laravel Queues (DB / Redis)**|معالجة المهام الثقيلة في الخلفية              |✅ أساسي         |
|**جدول `ai_cache`**            |تخزين إجابات AI المتكررة لتوفير التكلفة والوقت|✅ ذكي           |


> **نصيحة:** استخدم Redis في الإنتاج لأنه يدعم Queues و Sessions و Cache في آنٍ واحد. لا تستخدم Memcached و Redis معًا في نفس المشروع بدون مبرر واضح.

## 📌 ملاحظات عامة من المراجعة

> هذه التوصيات إضافية بناءً على مراجعة القائمة:

1. **تجنّب التكرار:** `Sentry` و`Laravel Telescope` و`Laravel Debugbar` تخدم أغراضًا مختلفة — Telescope و Debugbar للتطوير فقط، Sentry للإنتاج فقط. لا تُشغّلهم معًا في الإنتاج.
1. **Laravel Forge vs Envoyer:** Forge لإعداد الخادم، Envoyer للنشر. يمكن استخدامهما معًا أو استبدالهما بـ GitHub Actions مع scripts بسيطة.
1. **Rubix ML في PHP:** مناسب للنماذج الخفيفة، لكن للنماذج الثقيلة (LLMs) استخدم FastAPI بدلًا منه دائمًا.
1. **Redis أولًا:** إذا كنت تستخدم Redis، فلا حاجة لـ Memcached — Redis يغطي كل وظائفه ويزيد.
1. **L5 Swagger:** تأكد من تحديث التوثيق مع كل تغيير في الـ API، وإلا يصبح عبئًا لا ميزة.
